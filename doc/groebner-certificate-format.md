# Groebner Certificate Format

This is the first external-CAS import format used by `nodal-core`.

The purpose is deliberately narrow: an external system may compute a candidate
Groebner basis, but Rust verifies the certificate by exact rational arithmetic.
For a certificate with generators `F` and candidate basis `G`, the verifier checks:

```text
1. G is a Groebner basis by Buchberger's criterion:
   every S-polynomial reduces to 0 by G.
2. Every original generator in F reduces to 0 by G.
```

This proves that the imported basis is at least a verified Groebner basis for an
ideal containing the original generators. It can also compute the standard monomials
of the leading monomial ideal when the basis is visibly zero-dimensional, giving the
quotient algebra length. For a full chart certificate we still need the surrounding
projective chart/saturation and reducedness checks. When we need ideal equality,
degree05 adds a separate `.lift` certificate proving every imported basis element
lies in the original ideal.

## Syntax

The format is line-oriented and intentionally easy for external CAS scripts to emit.

```text
order lex

generators
poly
  1 2 0
  -1 0 1
end
poly
  1 1 1
  -1 0 0
end

basis
poly
  1 1 0
  -1 0 2
end
poly
  1 0 3
  -1 0 0
end
```

Supported monomial orders:

```text
lex
grlex
grevlex
```

Each term line has:

```text
coefficient exponent_0 exponent_1 ... exponent_{N-1}
```

For `N = 2`, the term:

```text
-3/2 4 1
```

means:

```text
(-3/2) * x0^4 * x1
```

Coefficients are rational numbers in either integer or `numerator/denominator`
form. Comments begin with `#`. Blank lines are ignored.

## Current Status And Limits

- The parser/verifier is generic over the coefficient field. Small fixtures still
  use `Rational` backed by `i128`; degree05 certificates can use `BigRational`
  backed by `num-bigint`/`num-rational`.
- The format does not yet encode chart metadata, saturation variables, quotient
  dimension expectations, radicality, or node-coordinate data.
- The verifier does not compute a Groebner basis; it verifies an imported one.
- Quotient dimension is computed only when each variable has a pure-power
  leading monomial bound. The result is a length with multiplicity, not by itself a
  proof that the points are reduced.

## Degree05 Role

For the Togliatti quintic, the intended certificate stack is:

```text
chart metadata
original Jacobian generators in that chart
external CAS candidate Groebner basis
Rust verification of Buchberger criterion and generator reduction
quotient-dimension/reducedness evidence
algebraic point certificates for ordinary nodes
```

This file documents only the imported basis layer. The projective and reducedness
layers are still part of the active `degree05` work.

Strict degree05 affine chart certificates now exist for all four charts:

```text
crates/degree05/certificates/togliatti-chart0-grevlex.cert
crates/degree05/certificates/togliatti-chart1-grevlex.cert
crates/degree05/certificates/togliatti-chart2-grevlex.cert
crates/degree05/certificates/togliatti-chart3-grevlex.cert
```

They are generated by Cygwin-based Singular from the Rust determinant model, then
parsed as `GroebnerCertificate<3, BigRational>`. The Rust tests verify that each
certificate's four original generators match `togliatti_affine_chart_generators(i)`,
that each imported basis satisfies Buchberger's criterion, and that every affine
chart quotient dimension is `16`.

Projective support strata certificates also exist:

```text
crates/degree05/certificates/togliatti-support-01-grevlex.cert
...
crates/degree05/certificates/togliatti-support-15-grevlex.cert
```

These use `GroebnerCertificate<4, BigRational>` because they add a localization
variable `tau`. They currently expose a model-level issue: the exact support-cover
certificate for the transcribed Proposition 130 determinant formula gives length
`16`, while the literature target remains `31`.

The active 31-point certificate line now uses the special Togliatti equation from
the same Catanese survey. Its certificates are parsed over
`QuadraticRational`, with Singular coefficients in `Q[t]/(t^2-5)`:

```text
crates/degree05/certificates/special-togliatti-chart3-grevlex.cert
crates/degree05/certificates/special-togliatti-chart3-hessian-bad-grevlex.cert
crates/degree05/certificates/special-togliatti-infinity-support-01-grevlex.cert
...
crates/degree05/certificates/special-togliatti-infinity-support-07-grevlex.cert
```

Rust verifies that the `w=1` chart has quotient length `31` and that the seven
support strata on `w=0` are empty. It also verifies that the Hessian-degenerate
locus inside the `w=1` singular locus is empty:

```text
<F, Fx, Fy, Fz, det Hess(F)> = <1>
```

Together these imported Groebner certificates prove, in Rust exact arithmetic,
that the special Togliatti model has `31` reduced ordinary nodes and no further
projective singular point. The format itself still does not encode arbitrary
saturation witnesses or algebraic point-coordinate data; those remain separate
future certificate layers.

For the special branch, each `.cert` now has a matching `.lift` file:

```text
crates/degree05/certificates/special-togliatti-chart3-grevlex.lift
crates/degree05/certificates/special-togliatti-chart3-hessian-bad-grevlex.lift
crates/degree05/certificates/special-togliatti-infinity-support-01-grevlex.lift
...
crates/degree05/certificates/special-togliatti-infinity-support-07-grevlex.lift
```

Each lift file stores coefficients `L[i,j]` for identities
`G[j] = sum_i L[i,j] * F[i]`. Rust verifies these identities over
`BigQuadraticRational`, because the Hessian-bad unit witness has coefficients too
large for the older `i128`-backed `QuadraticRational`. This closes the reverse
inclusion `G subset I`; the original normal-form check gives `I subset <G>`.
