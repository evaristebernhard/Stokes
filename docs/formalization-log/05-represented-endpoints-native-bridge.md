# 05. represented endpoints 的价值与 native bridge 的边界

## 数学阻塞

经典 Stokes 证明在局部化后得到的是一个有限和等式：

```text
Σ local bulk terms = Σ local true-boundary terms
```

纸面上，数学家随后把左边识别为 `∫_M dω`，右边识别为 `∫_{∂M} ω`。这个识别依赖某个全局 manifold-form integration API、boundary measure、orientation convention、chart-independence theorem。

当前项目的核心结果是 represented endpoint equality。它并不是“还没完成所以随便造两个 Real”。它是经典局部到整体证明展开后自然出现的有限坐标和。

## 错误或不自然路线

早期很多接口把 endpoint adapter、global integral interface、measure localization fields 作为大证书传递。这有两个问题：

1. 容易让人误以为 represented endpoint 是任意命名的两个 `Real` 字段；
2. 容易掩盖 local finite-sum reconstruction 本身也是一个数学步骤。

`reports/stokes-m8-field-reduction.md` 和 `reports/strict_buffer_to_stokes_audit.md` 都记录过类似状态：很多 measure/boundary/orientation 层当时仍 fieldized，但 finite-sum localization、artificial-face cancellation、boundary COV transport 等局部组件已经有 theorem-backed 进展。

后来的 FirstPrinciples route 采取更干净的策略：canonical represented endpoints 由 intrinsic route 生成，不作为用户输入。

## 最终解决方案

endpoint algebra 层在：

```text
Stokes/Global/CoverIndexedLocalizedRefinedEndpoint.lean
```

关键结构：

```lean
CoverIndexedBoundaryLocalizedRefinedPartition
CoverIndexedBoundaryLocalizedRefinedPartition.generatedRepresentedBulkIntegral
CoverIndexedBoundaryLocalizedRefinedPartition.generatedRepresentedBoundaryIntegral
CoverIndexedBoundaryLocalizedRefinedPartition.representedStokes_of_localStokes
```

这个结构故意很小：它只记录 refined boundary pieces、local bulk terms、local boundary terms。于是 represented bulk/boundary integral 是定义性生成的 nested finite sums：

```text
generatedRepresentedBulkIntegral
  = Σ i, Σ q ∈ boundaryPieces i, localBulkTerm i q

generatedRepresentedBoundaryIntegral
  = Σ i, Σ q ∈ boundaryPieces i, localBoundaryTerm i q
```

zero-localized certificate 层在：

```text
Stokes/Global/CoverIndexedZeroLocalizedIntrinsicCertificate.lean
```

关键 API：

```lean
CoverIndexedBoundaryLocalizedRefinedPartition.ZeroLocalStokesData
CoverIndexedBoundaryLocalizedRefinedPartition.ZeroLocalStokesData.localStokes_of_interiorFields
CoverIndexedBoundaryLocalizedRefinedPartition.boundaryHalfSpaceBulkSum_eq_trueBoundarySum_of_zeroLocalStokesData
CoverIndexedBoundaryLocalizedRefinedPartition.representedStokes_of_zeroLocalStokesData
CoverIndexedBoundaryLocalizedRefinedPartition.ZeroLocalizedEndpointCertificate
CoverIndexedBoundaryLocalizedRefinedPartition.ZeroLocalizedEndpointCertificate.representedStokes
```

这里的逻辑是：

```text
per-box zero support + per-box interior fields
⇒ each localBulkTerm = localBoundaryTerm
⇒ nested finite sum equality
⇒ generated represented endpoint equality
```

最后 intrinsic route 选择 canonical endpoints：

```lean
CoverIndexedZeroCompactRepresentedStokesIntrinsicInput.canonicalRepresentedBulkIntegral
CoverIndexedZeroCompactRepresentedStokesIntrinsicInput.canonicalRepresentedBoundaryIntegral
```

FirstPrinciples route 再投影为：

```lean
CoverIndexedZeroCompactRepresentedStokesFirstPrinciplesInput.canonicalRepresentedBulkIntegral
CoverIndexedZeroCompactRepresentedStokesFirstPrinciplesInput.canonicalRepresentedBoundaryIntegral
```

## represented theorem 与 native theorem 的精确差别

represented theorem 已经证明：

```text
the finite coordinate sum generated by local Stokes on dω
=
the finite coordinate boundary sum generated by true lower half-space faces
```

尚未声称：

```text
that finite coordinate bulk sum = a chosen native manifold integral ∫_M dω
that finite coordinate boundary sum = a chosen native boundary integral ∫_{∂M} ω
```

这个差别不是“有没有 Stokes 内容”的差别，而是“local-to-global Stokes proof”和“native manifold integration API”的分层。

`Stokes/Global/GlobalIntegralDefinitions.lean` 与 `Stokes/Global/CanonicalIntegralInterface.lean` 记录了 represented global values 和 canonical interface 的命名层。`reports/remaining_input_scoreboard.md` 也明确把最终出口称为 represented statement through `CanonicalIntegralInterface`，而不是 mathlib-native manifold integral statement。

## 为什么这仍然是可发表核心结果

represented Stokes 已经包含以下核心数学 payload：

- compact support 产生有限局部化路线；
- 每个局部 piece 满足 half-space local Stokes；
- artificial faces 被 zero-localized support 杀掉；
- lower normal faces 组成 represented boundary endpoint；
- finite local equalities 重构 canonical represented endpoint equality；
- theorem input 被压缩到 first-principles smoothness + compact support。

这已经远远超过“局部 Stokes lemma”。它形式化的是经典证明中从紧支撑到有限坐标和等式的主体部分。native bridge 是下一篇或下一阶段工作的自然接口。

## 对主 theorem 的贡献

主 theorem：

```lean
CoverIndexedZeroCompactRepresentedStokesFirstPrinciplesInput.representedStokes
```

最终证明的是：

```lean
X.canonicalRepresentedBulkIntegral =
  X.canonicalRepresentedBoundaryIntegral
```

这两个 endpoint 不是输入字段，而是由 intrinsic route 生成。这个设计让 theorem 有真实数学内容：它不从用户手里拿 endpoint equality，而是从局部 Stokes 构造出来。

## 论文可用表述

中文表述：

> represented endpoint 是 Stokes 经典证明局部化后的有限坐标和，而不是一个临时占位符。本项目证明了这些由 compact support 和 chartwise smoothness 自动生成的有限坐标端点相等。剩余的 native bridge 是将该有限坐标和识别为某个 manifold integral API 的工作，概念上位于 Stokes 局部到整体证明之上。

英文表述：

> The represented endpoints are not placeholders. They are the canonical finite coordinate sums produced by the local-to-global proof of Stokes. The formal theorem proves equality of these generated endpoints; the remaining native bridge is the comparison between these finite coordinate sums and a chosen manifold integration interface.
